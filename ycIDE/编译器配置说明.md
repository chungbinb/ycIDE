# 编译器配置说明

ycIDE 使用 LLVM/Clang 作为编译后端，搭配 MSVC SDK（Windows SDK + MSVC 运行时）提供头文件和库。

> **注意**：由于 LLVM 和 MSVC SDK 文件较大，不包含在 GitHub 仓库中，需要手动下载配置。
> 正式发布版本会自带精简后的编译器和运行时，用户开箱即用。

## 配置步骤

### 第一步：下载 LLVM/Clang（编译器）

- 下载地址: https://github.com/llvm/llvm-project/releases
- 下载文件: `clang+llvm-22.1.0-x86_64-pc-windows-msvc.tar.xz`（绿色免安装版，约 822 MB）
- 使用 7-Zip 解压（需要解两次：`.tar.xz` → `.tar` → 文件夹）
- 将解压出来的内容放到 `ycIDE程序目录\compiler\llvm\` 下

### 第二步：配置 MSVC SDK（头文件和库）

从本机 Visual Studio 2022 安装目录复制，或从正式发布包解压：

**MSVC 运行时**（C++ 标准库头文件 + 运行时库）：
- 来源: `<VS安装目录>\VC\Tools\MSVC\14.44.35207\`
- 复制 `include\` 和 `lib\x86\`、`lib\x64\`（跳过 PDB、store、uwp 子目录）
- 放到 `ycIDE程序目录\compiler\MSVCSDK\MSVC\14.44.35207\` 下

**Windows SDK**（Win32 API 头文件 + 导入库）：
- 来源: `C:\Program Files (x86)\Windows Kits\10\`
- 复制 `Include\10.0.26100.0\` 下的 `ucrt`、`um`、`shared` 子目录
- 复制 `Lib\10.0.26100.0\` 下的 `ucrt\x86`、`ucrt\x64`、`um\x86`、`um\x64`
- 放到 `ycIDE程序目录\compiler\MSVCSDK\WindowsKits\10\` 下

### 最终目录结构

```
ycIDE程序目录/
├── ycIDE.exe
├── compiler/
│   ├── llvm/                              # LLVM/Clang 编译器
│   │   └── bin/
│   │       ├── clang.exe                  (编译器)
│   │       ├── lld-link.exe               (MSVC兼容链接器)
│   │       └── ...其他文件
│   └── MSVCSDK/                           # MSVC SDK
│       ├── MSVC/14.44.35207/
│       │   ├── include/                   (vcruntime.h, vector, string 等)
│       │   └── lib/
│       │       ├── x86/                   (libcmt.lib, msvcrt.lib 等)
│       │       └── x64/
│       └── WindowsKits/10/
│           ├── Include/10.0.26100.0/
│           │   ├── ucrt/                  (stdio.h, stdlib.h 等)
│           │   ├── um/                    (windows.h, winuser.h 等)
│           │   └── shared/                (windef.h 等)
│           └── Lib/10.0.26100.0/
│               ├── ucrt/x86/, ucrt/x64/   (UCRT 库)
│               └── um/x86/, um/x64/       (kernel32.lib, user32.lib 等)
└── ...其他ycIDE文件
```

## 编译器查找顺序

Clang 编译器按以下顺序查找：

1. `程序目录\compiler\llvm\bin\clang.exe`
2. `程序目录\compiler\bin\clang.exe`
3. `程序目录\llvm\bin\clang.exe`
4. 系统 PATH 环境变量中的 clang.exe

MSVC SDK 按以下顺序查找：

1. `程序目录\compiler\MSVCSDK`
2. `程序目录\MSVCSDK`

版本号子目录会自动检测，无需硬编码。

## 分发时精简（正式发布用）

开发阶段使用完整包即可。正式发布时可精简：

**LLVM**（精简后约 150 MB）— 只保留 `bin/` 下的：
- `clang.exe`、`lld-link.exe`、`LLVM-C.dll`、`libclang.dll`、`LTO.dll`、`Remarks.dll`
- 可选保留 `llvm-strip.exe`（用于剥离调试符号减小体积）
- 删除其他 100+ 个 exe 工具和 `lib/`、`include/`、`share/` 等目录

**MSVC SDK**（精简后约 300-500 MB）— 可删除：
- `lib/` 下的所有 `.pdb` 文件
- `lib/` 下的 `store/`、`uwp/` 子目录
- `Include/` 下的 `winrt/`、`cppwinrt/` 子目录（除非需要 WinRT）
- MSVC 下的 `crt/src/` 目录（CRT 源码，不需要）

## 工作原理

```
.eyc 源代码 → 转换为 C 代码 → Clang 编译 → .exe 可执行文件
                                  ↑
                       使用 MSVC SDK 提供的
                       windows.h 和 kernel32.lib
```

编译目标: `--target=x86_64-pc-windows-msvc`（x86 时为 `i686-pc-windows-msvc`）
链接器: `lld-link`（MSVC 兼容模式，支持 .lib 格式）

## 验证安装

启动 ycIDE，创建或打开一个项目，点击工具栏上的"运行"按钮。
如果在输出窗口看到"正在编译..."并成功生成可执行文件，说明编译器已正确配置。

## 常见问题

**Q: 为什么需要两个包（LLVM + MSVC SDK）？**
A: LLVM/Clang 只是编译器，不自带 Windows 头文件和库。MSVC SDK 提供了 Windows API 头文件、导入库和 C/C++ 运行时库。

**Q: 编译报错 `windows.h file not found`？**
A: MSVC SDK 未正确配置。确保 `compiler\MSVCSDK\WindowsKits\10\Include\10.0.26100.0\um\windows.h` 文件存在。

**Q: 编译报错 `找不到Clang编译器`？**
A: LLVM 未正确配置。确保 `compiler\llvm\bin\clang.exe` 文件存在。

**Q: 编译报错 `找不到 MSVC SDK`？**
A: 确保 `compiler\MSVCSDK\` 目录结构完整，包含 `MSVC` 和 `WindowsKits` 两个子目录。

**Q: 用户需要安装 Visual Studio 吗？**
A: 不需要。MSVC SDK 已打包在程序目录中，完全绿色免安装。
