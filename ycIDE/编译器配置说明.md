# 编译器配置说明

ycIDE 使用 LLVM/Clang 作为编译后端，搭配 MinGW-w64 提供的 Windows 头文件和库。

> **注意**：由于 LLVM 和 MinGW 文件较大，不包含在 GitHub 仓库中，需要手动下载配置。
> 正式发布版本会自带精简后的编译器和运行时，用户开箱即用。

## 配置步骤

### 第一步：下载 LLVM/Clang（编译器）

- 下载地址: https://github.com/llvm/llvm-project/releases
- 下载文件: `clang+llvm-22.1.0-x86_64-pc-windows-msvc.tar.xz`（绿色免安装版，约 822 MB）
- 使用 7-Zip 解压（需要解两次：`.tar.xz` → `.tar` → 文件夹）
- 将解压出来的内容放到 `ycIDE程序目录\compiler\llvm\` 下

### 第二步：下载 MinGW-w64（Windows 头文件和库）

- 下载地址: https://github.com/niXman/mingw-builds-binaries/releases
- 下载文件: `x86_64-15.2.0-release-posix-seh-ucrt-rt_v13-rev1.7z`（约 98 MB）
- 解压后将 `mingw64` 文件夹放到 `ycIDE程序目录\compiler\` 下

### 最终目录结构

```
ycIDE程序目录/
├── ycIDE.exe
├── compiler/
│   ├── llvm/                          # LLVM/Clang 编译器
│   │   └── bin/
│   │       ├── clang.exe              (编译器)
│   │       ├── lld-link.exe           (链接器)
│   │       └── ...其他文件
│   └── mingw64/                       # MinGW 运行时（头文件+库）
│       ├── x86_64-w64-mingw32/
│       │   ├── include/
│       │   │   └── windows.h          (Windows API 头文件)
│       │   └── lib/
│       │       └── libkernel32.a      (Windows API 导入库)
│       ├── include/
│       └── lib/
└── ...其他ycIDE文件
```

## 编译器查找顺序

Clang 编译器按以下顺序查找：

1. `程序目录\compiler\llvm\bin\clang.exe`
2. `程序目录\compiler\bin\clang.exe`
3. `程序目录\llvm\bin\clang.exe`
4. 系统 PATH 环境变量中的 clang.exe

MinGW 运行时按以下顺序查找：

1. `程序目录\compiler\mingw64`
2. `程序目录\mingw64`

## 分发时精简（正式发布用）

开发阶段使用完整包即可。正式发布时可精简到约 150-200 MB：

**LLVM** — 只保留 `bin/` 下的：
- `clang.exe`、`lld-link.exe`、`LLVM-C.dll`、`libclang.dll`、`LTO.dll`、`Remarks.dll`
- 删除其他 100+ 个 exe 工具和 `lib/`、`include/`、`share/` 等目录

**MinGW** — 只保留：
- `x86_64-w64-mingw32/include/`（Windows API 头文件）
- `x86_64-w64-mingw32/lib/`（导入库）
- `include/`（C 标准库头文件）
- `lib/gcc/`（GCC 运行时库）
- 删除 `bin/`（gcc.exe 等不需要）、`share/`、`opt/` 等

## 工作原理

```
.eyc 源代码 → 转换为 C 代码 → Clang 编译 → .exe 可执行文件
                                  ↑
                         使用 MinGW 提供的
                         windows.h 和 libkernel32.a
```

编译目标: `--target=x86_64-w64-mingw32`（不依赖 Visual Studio 或 Windows SDK）

## 验证安装

启动 ycIDE，创建或打开一个项目，点击工具栏上的"运行"按钮。
如果在输出窗口看到"正在编译..."并成功生成可执行文件，说明编译器已正确配置。

## 常见问题

**Q: 为什么需要两个包（LLVM + MinGW）？**
A: LLVM/Clang 只是编译器，不自带 Windows 头文件和库。MinGW 提供了独立于微软 SDK 的 Windows API 头文件和导入库，这样用户不需要安装 Visual Studio。

**Q: 编译报错 `windows.h file not found`？**
A: MinGW 未正确配置。确保 `compiler\mingw64\x86_64-w64-mingw32\include\windows.h` 文件存在。

**Q: 编译报错 `找不到Clang编译器`？**
A: LLVM 未正确配置。确保 `compiler\llvm\bin\clang.exe` 文件存在。
